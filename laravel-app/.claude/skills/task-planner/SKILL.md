# タスク細分化・実装計画策定スキル

「$ARGUMENTS」を具体的な実装タスクに分解します。

## 実行手順

### Step 1: 要件の明確化
- 曖昧な点を質問で確認
- 「〇〇という理解で良いですか？」と1行で確認
- スコープを明確に定義

### Step 2: 影響範囲の調査
- 変更が必要なファイルを特定
- 既存コードとの依存関係を確認
- 影響を受けるコンポーネントを洗い出し

### Step 3: タスク分解
以下の粒度でタスクを分解：
- 1タスク = 1ファイルの修正 or 1機能の実装
- 依存関係を明示
- 優先順位を設定

### Step 4: 実装計画
```
## 実装計画

### Phase 1: [フェーズ名]
- [ ] タスク1（ファイル名）
- [ ] タスク2（ファイル名）

### Phase 2: [フェーズ名]
- [ ] タスク3（ファイル名）
```

## 出力形式

```
## 要件サマリー
[1-2行で要約]

## 影響ファイル
- path/to/file1.php - [変更内容]
- path/to/file2.blade.php - [変更内容]

## タスク一覧（依存順）

### 1. [タスク名]
- ファイル: path/to/file
- 内容: [具体的な変更内容]
- 依存: なし / タスクX完了後

### 2. [タスク名]
...

## 懸念点・確認事項
- [あれば記載]
```

## 注意事項（CLAUDE.mdより）
- MVP重視：最小限の構成で実現
- 勝手に実装しない：計画の承認を得てから実行
- 余計な機能追加は提案しない
