# 技術記事・ドキュメント執筆スキル

$ARGUMENTS について執筆をサポートします。

## 実行手順

### Step 1: 企画フェーズ
- 対象読者を確認
- 記事のゴール（読者が得られること）を明確化
- 必要な前提知識を整理

### Step 2: 構成案作成
以下の形式で構成案を提示：
```
1. 導入（課題提起）
2. 解決策の概要
3. 具体的な実装・手順
4. まとめ
```

### Step 3: 執筆
- 各セクションを順番に執筆
- コードサンプルは実際に動作するものを提示
- 図表が必要な箇所を指摘

### Step 4: 推敲
- 冗長な表現の削除
- 技術的な正確性の確認
- 読みやすさの改善

## 出力形式
- Markdown形式
- 見出しレベルは適切に使用
- コードブロックには言語指定を付与
